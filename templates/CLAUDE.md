## 会话启动检查清单

每次会话开始时，必须按顺序执行：

1. `pwd` - 确认工作目录
2. `git log --oneline -5` - 查看最近提交
3. `cat claude-progress.txt` - 阅读进度文件
4. `cat feature-list.json` - 阅读功能清单
5. `bash agent-relay/scripts/init-dev.sh` - 启动开发环境
6. 确认开发服务器正常运行
7. 根据功能选择规则确定本次任务

## 核心规则

1. **一次一个功能**：每次会话只实现一个功能
2. **E2E 测试**：每个功能必须通过端到端测试
3. **小步提交**：完成一个功能就提交一次
4. **更新进度**：每次提交后更新进度文件

## 开发流程

```
启动会话 → 阅读进度 → 选择功能 → 实现功能 → E2E 测试 → Git 提交 → 更新进度 → 结束会话
```

## Git 提交规范

- `feat(FXXX): 功能描述` - 新功能
- `fix(FXXX): 修复描述` - Bug 修复
- `test(FXXX): 测试描述` - 测试相关

## 进度文件规则

- `claude-progress.txt` 是会话间的交接文档
- 每次会话结束前必须更新
- 记录：已完成的功能、遇到的问题、做出的决策、下一步建议

## 禁止行为

1. 不要删除或修改 `test_criteria`
2. 不要将未测试的功能标记为完成
3. 不要跳过启动检查清单
4. 不要不提交代码就结束会话
5. 不要同时实现多个功能
6. 不要修改其他功能的代码（除非是必要的依赖修复）
